## Направления улучшений

### 1. Stop-words фильтрация (РЕАЛИЗОВАНО)
Базовый TF-IDF учитывает служебные слова ("the", "is", "a", "of"), которые
не несут смысла, но раздувают словарь и снижают качество поиска.
**Решение:** добавить параметр `stop_words="english"` в TfidfVectorizer.

### 2. Dense-эмбеддинги
TF-IDF ищет по совпадению слов и не понимает синонимы и парафразы.
**Решение:** заменить TF-IDF на sentence-transformers
(например `intfloat/multilingual-e5-small`) — поиск по смыслу.

### 3. Гибридный поиск
Совмещение TF-IDF (лексический сигнал) и dense (семантический сигнал)
обычно даёт лучше recall, чем любой из них по отдельности.
**Решение:** считать оба score, нормализовать, взвешенно складывать
(например 0.4 * tfidf + 0.6 * dense).

### 4. Лемматизация для русскоязычных корпусов
TF-IDF не находит формы одного слова ("ипотека" vs "ипотечный").
**Решение:** прогонять тексты через pymorphy2 перед векторизацией.

### 5. Query expansion (расширение запроса)
Короткие запросы плохо матчатся.
**Решение:** перед поиском добавлять синонимы из словаря или
генерировать парафразы через LLM (HyDE).

### 6. Re-ranking
После retrieval отбирать top-20, потом переранжировать
через cross-encoder (например `BAAI/bge-reranker-base`) и брать top-5.
Это медленнее, но точнее.

### 7. Метрики качества (recall@k, MRR)
Сейчас нет автоматической оценки качества поиска.
**Решение:** разметить ~20 пар (вопрос → ожидаемый doc_id),
посчитать recall@5 и MRR. Это позволит сравнивать улучшения объективно.

### 8. Метаданные-фильтры
Сейчас поиск идёт по всему корпусу.
**Решение:** добавить фильтры по `source` / категории / дате в UI.
